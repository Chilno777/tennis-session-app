# Session Design

## なぜSessionを作ったか

スコアを練習会ごとに独立させるため。

従来はグローバルな状態だったため、
練習をまたぐとデータが混在していた。

## 改善点

- データの独立性向上
- DB移行が容易
- 状態管理の明確化

## セッション導入前の問題点

・スコアがグローバル管理されていた
・練習をまたぐとデータが混ざる

## 導入後の変化

SessionがMatchのリストを保持することで、
練習単位で完全に分離できるようになった。
